# 静态简历网页需求文档

## 1. 项目目标
- 构建一个基于原生 `HTML/CSS/JS` 的静态简历网页，用于在线展示个人简历。
- 页面视觉采用 A4 纸张布局，强化“像正式简历”的阅读体验。
- 页面优先面向桌面端浏览，兼顾基础打印能力，但不以打印导出为首要目标。
- 通过右下角悬浮工具栏提供简历预览缩放能力，提升不同屏幕尺寸下的阅读舒适度。

## 2. 用户场景
### 2.1 主要用户
- 求职者本人：维护和分享在线简历。
- 招聘方 / 面试官：在浏览器中快速查看简历信息。
- 熟人推荐场景中的查看者：通过链接直接阅读简历内容。

### 2.2 核心使用场景
- 在电脑浏览器中打开简历链接并进行阅读。
- 根据屏幕大小点击右下角按钮放大查看内容细节。
- 在保持 A4 版式不破坏的前提下浏览完整简历信息。
- 后续手动替换占位文案为真实简历内容并继续维护。

## 3. 产品定位与范围
### 3.1 产品定位
- 一个轻量、正式、便于部署的单页静态简历站点。
- 强调信息清晰、排版稳定、部署简单，不追求复杂动画和后台能力。

### 3.2 本期范围
- 单页静态简历页面。
- A4 视觉布局。
- 标准简历信息模块。
- 右下角悬浮缩放工具栏。
- 浏览器直接打开即可访问。

### 3.3 非本期范围
- 后台编辑系统。
- 多主题切换。
- 多语言支持。
- PDF 导出功能。
- 自动分页、多页简历管理。
- 数据接口、登录系统、服务端渲染。

## 4. 信息架构
页面内容采用单页线性结构，自上而下展示：
1. 个人信息区
2. 个人简介
3. 工作 / 实习经历
4. 项目经历
5. 核心技能
6. 教育经历
7. 补充信息
8. 求职意向

## 5. 页面结构设计
### 5.1 页面容器
- 页面外层为整体浏览容器，用于负责背景、居中布局和整体滚动。
- 页面中层为缩放舞台容器，用于承接缩放后的宽高占位，保证页面放大后仍保持居中和可滚动。
- 页面内层为实际简历卡片，使用固定 A4 比例呈现。

### 5.2 模块设计
#### 个人信息区
- 展示姓名、岗位方向、所在地、电话、邮箱、代码仓库或个人主页。
- 左侧突出姓名与岗位，右侧展示联系信息。

#### 个人简介
- 用一段简短总结介绍候选人的经验、能力和优势。
- 控制在 2 到 4 行，避免冗长。

#### 工作 / 实习经历
- 使用时间轴式块状结构展示公司、职位、时间、职责与结果。
- 每段经历以 2 到 4 条要点概括，强调动作和结果。

#### 项目经历
- 展示项目名称、技术栈、项目时间和关键贡献。
- 重点写清业务场景、负责内容和成果。

#### 核心技能
- 使用标签化展示，突出技术关键词。
- 避免长段落描述，优先可扫读。

#### 教育经历
- 展示学校、专业、学历、时间。

#### 补充信息
- 可放英语水平、协作能力、证书、开源经历等补充信息。

#### 求职意向
- 展示岗位方向、城市偏好、到岗状态等信息。

## 6. 视觉规范
### 6.1 整体风格
- 风格关键词：专业、极简、正式、清晰。
- 使用浅色背景承托白色纸张卡片。
- 采用单一蓝色作为强调色，用于模块标题标识和局部点缀。

### 6.2 A4 版式规则
- 简历主体遵循 `210mm × 297mm` 的 A4 比例。
- 页面卡片具有圆角、阴影和留白边距，增强纸张观感。
- 默认在桌面视口中水平居中展示。
- 缩放时保持简历从顶部中线向两侧放大，避免顶部内容被裁出可视区域。

### 6.3 字体与间距
- 默认字体使用系统中文友好字体组合。
- 标题、正文、辅助信息采用固定字号层级。
- 行高与模块间距保持稳定，保证阅读节奏一致。

### 6.4 滚动条策略
- 页面只保留一套浏览器主滚动条，避免重复滚动区域。
- 滚动条采用更细、更柔和的视觉样式，减少对简历内容的干扰。

## 7. 交互规则
### 7.1 悬浮工具栏
- 固定在页面右下角。
- 当前采用横向排布，包含：
  - 放大
  - 缩小
  - 当前缩放比例 / 重置
- 工具栏风格为半透明浅色浮层，不应抢占正文视觉焦点。

### 7.2 缩放规则
- 缩放仅作用于简历卡片预览，不改变真实内容结构。
- 默认缩放比例为 `100%`。
- 最小缩放比例为 `100%`，不允许缩得更小。
- 最大缩放比例为 `130%`。
- 每次点击按 `10%` 步进变化。
- 当缩放达到上下限时，对应按钮进入禁用状态。

### 7.3 缩放时的滚动行为
- 缩放过程中应尽量保持用户当前阅读区域不发生剧烈跳变。
- 横向保持页面中心稳定。
- 纵向尽量保持当前阅读位置，避免每次缩放都回到页面顶部。

### 7.4 无 JavaScript 情况
- 即使脚本失效，静态简历正文仍然可以正常阅读。
- 仅缩放功能不可用，不影响页面内容可见性。

## 8. 技术方案
### 8.1 技术栈
- `index.html`：页面结构与内容。
- `styles.css`：A4 布局、视觉样式、滚动条样式、工具栏样式。
- `main.js`：缩放逻辑、按钮状态、滚动位置保持。

### 8.2 实现约束
- 使用原生 `HTML/CSS/JS` 实现，不引入前端框架。
- 不依赖后端接口。
- 不依赖构建工具。
- 可直接部署到任意静态托管平台。

### 8.3 关键实现约定
- 简历主节点使用 `.resume-page`。
- 缩放值通过 CSS 自定义变量 `--page-scale` 传递。
- 缩放后的舞台占位通过 `--stage-width` 和 `--stage-height` 协调。
- 工具栏按钮通过 `data-action` 标记行为，例如 `zoom-in`、`zoom-out`、`reset`。

## 9. 内容维护规则
- 当前页面中的简历内容为占位示例，需要后续替换为真实信息。
- 内容应尽量控制在单页 A4 内；若超长，优先精简文案，而不是破坏整体版式。
- 每段经历建议优先突出结果、数据、影响范围与技术贡献。
- 联系方式、链接和岗位意向应保持最新状态。

## 10. 非功能要求
### 10.1 性能
- 首屏应为纯静态资源加载。
- 页面资源结构简单，加载迅速。

### 10.2 兼容性
- 重点保证 Chromium 系浏览器下表现一致。
- 兼容常见桌面浏览器的现代 CSS 与基础 JavaScript 能力。

### 10.3 可访问性
- 工具按钮应包含可识别文本或 `aria-label`。
- 保持正文与背景的对比度，确保长时间阅读舒适。

### 10.4 可维护性
- 代码结构应清晰分离：结构、样式、行为分别维护。
- 文案替换时不应影响核心样式与交互逻辑。

## 11. 验收标准
### 11.1 页面展示
- 页面打开后，简历主体在桌面端水平居中。
- A4 卡片比例稳定，背景、阴影、圆角展示正常。
- 各模块层次清晰，无重叠、错位、溢出。

### 11.2 交互行为
- 右下角工具栏始终可见。
- 点击放大按钮后，缩放比例按 `10%` 增长，最大到 `130%`。
- 点击缩小按钮后，最小只能回到 `100%`。
- 点击重置按钮后，比例恢复到 `100%`。
- 缩放过程中页面不会跳到顶部，也不会出现双滚动条。

### 11.3 浏览体验
- 页面滚动顺畅。
- 滚动条样式统一、轻量、不突兀。
- 页面在无脚本时依然可读。
